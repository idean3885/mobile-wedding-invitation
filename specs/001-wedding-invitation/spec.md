# Feature Specification: 모바일 청첩장

**Feature Branch**: `001-wedding-invitation`
**Created**: 2026-03-22
**Status**: Draft
**Input**: User description: "모바일 청첩장 - 대문 사진/인사말, 갤러리, 오시는 길(지도), 마음 전하실 곳(계좌), 섹션 클릭 이동, OG 메타 태그(링크 공유 썸네일). 단일 페이지 스크롤 구조, 정적 사이트, 라운드트립 없음."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 청첩장 열람 (Priority: P1)

하객이 카카오톡/문자로 받은 링크를 탭하면 청첩장이 열린다. 대문 사진, 양가 부모 이름, 신랑·신부 이름과 서열, 예식 일시·장소, 인사말이 한 화면에 보이고, 아래로 스크롤하여 전체 내용을 볼 수 있다.

**Why this priority**: 청첩장의 핵심 목적. 이것 없이는 다른 기능이 의미 없음.

**Independent Test**: 링크를 모바일 브라우저에서 열어 대문 섹션이 정상 표시되고 스크롤이 동작하는지 확인.

**Acceptance Scenarios**:

1. **Given** 하객이 공유 링크를 받았을 때, **When** 링크를 탭하면, **Then** 대문 사진, 양가 부모 이름, 신랑·신부 이름과 서열, 예식 일시·장소, 인사말이 화면에 표시된다.
2. **Given** 청첩장이 로딩되었을 때, **When** 아래로 스크롤하면, **Then** 갤러리, 오시는 길, 마음 전하실 곳 섹션이 순서대로 나타난다.

---

### User Story 2 - 링크 공유 시 썸네일 미리보기 (Priority: P1)

하객이 카카오톡이나 메신저에서 청첩장 링크를 공유하면, 대표 사진과 "OO & OO 결혼합니다" 같은 제목이 썸네일로 표시된다.

**Why this priority**: 링크 공유는 청첩장 전달의 핵심 수단. 썸네일이 없으면 클릭률이 크게 떨어짐.

**Independent Test**: 카카오톡 채팅에 링크를 붙여넣어 썸네일 카드가 올바르게 표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 청첩장 URL이 있을 때, **When** 카카오톡에 링크를 붙여넣으면, **Then** 대표 사진, 제목("OO & OO 결혼합니다"), 예식 일시가 포함된 썸네일 카드가 표시된다.
2. **Given** 청첩장 URL이 있을 때, **When** 일반 메신저/SNS에 공유하면, **Then** 링크 미리보기(대표 사진, 제목, 설명)가 정상 표시된다.

---

### User Story 3 - 갤러리 열람 (Priority: P2)

하객이 갤러리 섹션에서 신랑·신부의 사진을 감상한다. 사진을 탭하면 크게 볼 수 있다.

**Why this priority**: 청첩장의 감성적 요소. 대문 다음으로 하객이 가장 기대하는 콘텐츠.

**Independent Test**: 갤러리 섹션에서 사진 목록이 표시되고, 탭 시 확대 뷰가 동작하는지 확인.

**Acceptance Scenarios**:

1. **Given** 갤러리 섹션에 도달했을 때, **When** 화면을 보면, **Then** 사진 목록이 표시된다.
2. **Given** 갤러리에서 사진을 탭했을 때, **When** 사진이 선택되면, **Then** 해당 사진이 전체 화면으로 확대되고 다른 사진으로 이동할 수 있다.
3. **Given** 확대된 사진을 보고 있을 때, **When** 확대 뷰를 닫으면, **Then** 갤러리 목록으로 돌아간다.

---

### User Story 4 - 오시는 길 확인 (Priority: P2)

하객이 오시는 길 섹션에서 예식장 위치와 교통 안내를 확인한다. 지도를 탭하면 외부 지도 앱으로 이동할 수 있다.

**Why this priority**: 실용적 핵심 정보. 하객이 길을 찾는 데 필수.

**Independent Test**: 오시는 길 섹션에서 지도 이미지와 주소가 표시되고, 탭 시 외부 지도 앱이 열리는지 확인.

**Acceptance Scenarios**:

1. **Given** 오시는 길 섹션에 도달했을 때, **When** 화면을 보면, **Then** 예식장 이름, 주소, 지도 이미지, 교통 안내(지하철/버스/자가용)가 표시된다.
2. **Given** 지도 영역을 탭했을 때, **When** 탭하면, **Then** 외부 지도 앱으로 이동하여 예식장 위치가 표시된다.
3. **Given** 지도 앱이 설치되지 않았을 때, **When** 지도를 탭하면, **Then** 웹 지도 페이지가 브라우저에서 열린다.

---

### User Story 5 - 마음 전하실 곳 (Priority: P3)

하객이 축의금을 보내기 위해 신랑측·신부측 계좌 정보를 확인하고 복사한다.

**Why this priority**: 부가 기능이지만 하객 편의에 중요. 현장 참석이 어려운 하객에게 특히 필요.

**Independent Test**: 마음 전하실 곳 섹션에서 계좌 정보가 표시되고 복사 기능이 동작하는지 확인.

**Acceptance Scenarios**:

1. **Given** 마음 전하실 곳 섹션에 도달했을 때, **When** 화면을 보면, **Then** 신랑측/신부측으로 구분되고 각 측별 본인·아버지·어머니 계좌 정보(은행명, 예금주, 계좌번호)가 표시된다.
2. **Given** 계좌 정보가 표시되었을 때, **When** 하객이 계좌번호 복사를 요청하면, **Then** 계좌번호가 클립보드에 복사되고 복사 완료 피드백이 표시된다.

---

### Edge Cases

- 이미지가 로딩되지 않을 때: 대체 텍스트가 표시되어야 한다.
- 매우 느린 네트워크(3G): 이미지가 점진적으로 로딩되고, 텍스트 콘텐츠가 먼저 표시되어야 한다.
- 클립보드 복사가 지원되지 않는 브라우저: 계좌번호를 텍스트 선택할 수 있도록 대체 방안을 제공한다.
- ~~가로 모드/데스크톱~~: 대응하지 않음. 모바일 세로(portrait)만 지원.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 단일 페이지 스크롤 구조로 모든 섹션을 MUST 제공한다.
- **FR-002**: 대문 섹션에 대표 사진, 양가 부모 이름, 신랑·신부 이름과 서열, 예식 일시, 예식장 이름, 인사말을 MUST 표시한다.
- **FR-003**: 갤러리 섹션에서 사진을 탭하면 확대 뷰를 MUST 제공한다.
- **FR-004**: 오시는 길 섹션에서 예식장 주소, 지도 이미지, 교통 안내를 MUST 표시한다.
- **FR-005**: 지도 영역 탭 시 외부 지도 앱으로 MUST 연결한다.
- **FR-006**: 마음 전하실 곳 섹션에서 계좌번호 복사 기능을 MUST 제공한다.
- **FR-007**: 링크 공유 시 썸네일 메타 정보(제목, 설명, 대표 사진)를 MUST 포함하여 메신저/SNS에서 미리보기가 표시되어야 한다.
- **FR-008**: 모든 이미지에 대체 텍스트를 MUST 포함한다.
- **FR-009**: 서버 라운드트립 없이 정적 파일만으로 MUST 동작한다.

### Key Entities

- **Wedding(예식 정보)**: 신랑 이름(동영), 신부 이름(지혜), 양가 부모 이름, 서열, 예식 일시(2026-06-06 토 15:00), 예식장(남부터미널 더화이트베일 3층 V홀), 대표 사진(IMG_4689.JPG), 인사말
- **Gallery(갤러리)**: 사진 목록(이미지 원본, 대체 텍스트, 순서)
- **Venue(오시는 길)**: 예식장 이름, 주소, 지도 이미지, 지도 앱 연결 링크, 교통 안내(지하철, 버스, 자가용)
- **Account(계좌 정보)**: 구분(신랑측/신부측), 관계(본인/아버지/어머니), 은행명, 예금주, 계좌번호. 측당 최대 3개(본인, 아버지, 어머니), 총 최대 6개

### Assumptions

- 청첩장 콘텐츠(사진, 텍스트, 계좌 정보)는 빌드 시점에 정적으로 포함된다.
- 지도는 빌드 시점에 생성된 정적 이미지로 제공한다. 런타임 외부 호출 없음.
- 지도 이미지 탭 시 외부 지도 앱으로 연결한다. 인라인 지도 임베드는 사용하지 않는다.
- 갤러리 사진은 13장이다 (실제 웨딩 사진 IMG_4677~4690 중 대문 제외).
- 대문 사진: IMG_4689.JPG
- 예식 일시: 2026년 6월 6일 토요일 오후 3시
- 예식장: 남부터미널 더화이트베일 3층 V홀
- 타겟 브라우저: iOS Safari 15+, Android Chrome 90+, Samsung Internet 15+.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 하객이 링크를 탭한 후 1.5초 이내에 대문 섹션이 표시된다 (4G 네트워크 기준).
- **SC-002**: 주요 메신저(카카오톡 포함)에서 링크 공유 시 썸네일 카드(대표 사진 최소 1200x630px + 제목 + 설명)가 정상 표시된다.
- **SC-003**: 갤러리 사진 탭 후 확대 뷰가 0.3초 이내에 표시된다.
- **SC-004**: 계좌번호 복사 후 0.5초 이내에 복사 완료 피드백이 표시된다.
- **SC-005**: 375px~430px 모바일 뷰포트에서 터치 타겟이 44x44px 이상이다.
